# Prompt Engineering – Structured Prompts

This folder contains structured, reusable prompts created as part of Week 2 – Prompt Engineering & API Automation.

All prompts are designed to be:
- API-ready
- Reusable
- Versioned
- Portfolio-friendly

---

## Prompt 1 – Manual Test Case to Automation Steps (Text)

**Goal:**  
Convert manual test cases into clear, tool-agnostic automation steps.

**Use cases:**
- Test automation planning
- Converting legacy manual test cases
- Selenium / Playwright frameworks

---

## Prompt 2 – Manual Test Case to Automation Steps (JSON)

**File:** `test_case_to_steps_json.json`

**Goal:**  
Convert manual test cases into structured JSON output suitable for APIs and automation pipelines.

**Enhancements:**
- Locator hints
- Expected results
- Schema validation

---

## Prompt 3 – Logs to Root Cause Analysis (JSON)

**File:** `logs_to_root_cause.json`

**Goal:**  
Analyze automation or application logs and identify root cause with severity and confidence scoring.

**Use cases:**
- CI/CD debugging
- Test failure triage
- Production support analysis
